#### 需求
```
背景：
    得上班，办公室、会议室...得购物，商超、餐厅...得出行，坐飞机、地铁.....
    附近xxx有人确诊，他/她去过哪？啥时候？我当时在哪....

解决问题：
    普通用户通过软件了解每次外出sari接触风险，适当选择较小风险
    登陆用户通过记录日记方式，跟踪日记风险变化，并及时发现并采取防控措施
    通过累积用户外出行为，帮助权威机构寻找危险传播人群，优化防控特征

风险规避：
    仅对用户外出行为做接触可能性的判断，不做非专业猜测
    仅认证机构可上报疑似和确诊病例，普通用户不提供上报功能
    通过人工审核，所有上报对应政府官方公告

用户端功能：
    提供门户网站实时疫情链接，聚合有效信息源
    未登陆用户可查询单次外出的风险程度
    登陆用户可保存外出记录，并可跟踪记录的风险动态
    机构人员管理疑似/确诊病例

管理端：
    认证注册的机构，仅政府允许发布的机构可入驻
    审核机构上报的案例，确保与政府官网一致
    累积外出行为，定位易传播人群

技术实现：
    前后端分离，用户端h5页，admin端pc网页
    后端go语言 http api，前端ant mobile + react
    数据库mysql，缓存redis
    运维资源使用阿里/腾讯云
```

#### 启发
```
解释一下，为什么现在要少出门，请坚持到正月十五，不要出门。
我们把现在的人群分为四种人A、B、C、D。
A是有武汉接触史的，目前500万人，分散在全国各地，容易找出来，现在已经找出来很多了。
A在旅途中遇到过素未平生绝不认识的B，A不知道B的存在，B也不知道A的存在。
A到达目的地后接触了C，无论接触了多少个C，这些C都能够找到，官方已经对他们隔离了。
D是居家没有外出的安全市民。
现在最大的隐患是B，根本找不到，没有人知道谁是B，自己根本不知道自己是B。D只要出门，就有碰到B的可能性，自己也就是2B了。
因此现在每个人相互隔离的目的就是用时间来筛出B，潜伏期两周，两周之内，B纷纷发病，水落石出。直到所有的B都发完病，并隔离治疗，疫情才能结束。
因此现在采取的所有措施都是减少新B (2B) 的产生。
1918年美国的瘟疫采取的是同一种办法。
大家懂了吗?
这个讲的挺明白
```
